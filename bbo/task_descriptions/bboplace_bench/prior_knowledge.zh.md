# 领域先验知识

- MGO 并不是直接对原始坐标向量打分。它会先通过 wire-mask-guided 的贪心解码，把提案映射成一个合法 placement，所以看起来相近的两个向量，有时也可能对应到差异明显的最终布局。
- 根据论文描述，macro 会先按“与其相连模块总面积”从大到小排序，然后依次移动到能最小化增量 HPWL 的合法网格上；若存在并列最优网格，则选择最接近原始提案坐标的位置。
- 因此，高连接度 macro 的坐标通常比弱连接 macro 更重要；保留整体相对空间结构，往往比对每一维做互不相关的独立扰动更有效。
- API 边界上的搜索空间是连续的，但解码后的 placement 落在网格上，因此真实搜索面会带有离散和分段特性。能严格守边界、并在早期保持多样性的优化器，通常比过早陷入单一局部区域的方法更稳健。
- HPWL 只是下游芯片质量的 proxy，而不是完整设计目标。好的 MP-HPWL 往往有帮助，但它仍只是对后续 placement、routing、timing 质量的近似。
